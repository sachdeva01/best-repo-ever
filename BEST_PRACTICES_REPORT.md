### 🏗️ 12-Factor — Fix I: `sys.path.append` with Hardcoded Absolute Path
**Files:** `backend/routers/dashboard.py`, `backend/routers/expected_returns.py`

**Problem:** Both routers mutated `sys.path` with a hardcoded machine-specific path inside a live request handler, running on **every single HTTP request**.
```python
# BEFORE — ran on every request, breaks on any other machine
import sys
sys.path.append('/Users/ssachdeva/Documents/Claude/my-app/backend')
from routers.portfolio_allocation import TARGET_ALLOCATION, get_current_yield
```
**Fix:** Moved to module-level imports at the top of each file. Runs once at startup, works on any machine.
```python
# AFTER — module-level, machine-agnostic
from routers.portfolio_allocation import TARGET_ALLOCATION, get_current_yield
```

### 🛡️ Error Handling + 🏗️ 12-Factor (XI: Logs) — Fix II: `print()` replaced with `logging`
**Files:** `backend/routers/market_data.py`, `backend/routers/portfolio_allocation.py`, `backend/backup.py`

**Problem:** Diagnostic output used `print()` instead of the standard `logging` module. `print()` can't be filtered by log level, routed by log aggregators, or silenced in production without code changes.
```python
# BEFORE
print(f"Error fetching market data for {data_type}: {e}")
print(f"[prices] refreshed for slot {slot}: {results}")
print(f"[backup] Saved {dest} ({size_kb} KB)")
```
**Fix:** All three files now import `logging` and use `logger = logging.getLogger(__name__)`.
```python
# AFTER
logger.error("Error fetching market data for %s: %s", data_type, e)
logger.info("Prices refreshed for slot %s: %s", slot, results)
logger.info("Backup saved: %s (%.1f KB)", dest, size_kb)
```

## Issues Found — Not Auto-Fixed (Require Design Decision)

### 🎨 Style — `alert()` in Frontend (9 occurrences)
**Files:** `AccountForm.jsx`, `AccountList.jsx`, `HoldingsList.jsx`, `AccountsPage.jsx`, `ExpenseCategoryCard.jsx`, `OptimalAllocation.jsx`

Browser `alert()` blocks the UI thread, can't be styled, and breaks in server-side rendering. Should be replaced with an in-app toast/notification component.
```jsx
// BEFORE
alert('Failed to create account: ' + err.message)

// RECOMMENDED
// Use a toast notification system (e.g., react-hot-toast or a custom Toast component)
toast.error('Failed to create account: ' + err.message)
```
**Priority:** Medium — functional but poor UX.

### ⚡ Performance — Redundant DB Query for Constant Value
**File:** `backend/routers/expected_returns.py` → `calculate_expected_growth_rate(db)`

Called on every `GET /api/dashboard/quick-stats` request to return the constant `0.06`. The DB is hit to read a value that never changes at runtime.

**Recommended fix:**
```python
# Replace the DB call with the config value or a module constant
CONSERVATIVE_GROWTH_RATE = 0.06
```
**Priority:** Low — 1 extra DB query per request, currently negligible.

### 🏗️ 12-Factor (VI: Stateless Processes) — In-Memory Price Cache
**File:** `backend/routers/portfolio_allocation.py` — `_price_cache`, `_price_slot`, `_prices_last_fetched`

The price cache is stored as module-level variables. This works fine for a single-process deployment but will break under multi-worker or multi-instance deployments — each worker maintains its own independent cache, causing inconsistent prices across requests.

**Recommended fix for production:** Move the cache to Redis or a shared backing service. For current single-process dev use, this is acceptable.
**Priority:** Low now, High before horizontal scaling.

---

### 🔒 Security — No Rate Limiting on Auth Endpoints
**File:** `backend/routers/auth.py`

`POST /api/auth/login` has no rate limiting. An attacker can attempt unlimited password guesses. FastAPI doesn't include rate limiting by default.

**Recommended fix:** Add `slowapi` middleware:
```python
from slowapi import Limiter
from slowapi.util import get_remote_address
limiter = Limiter(key_func=get_remote_address)

@router.post("/login")
@limiter.limit("10/minute")
def login(request: Request, body: LoginRequest, ...):
```
**Priority:** Medium — not critical for local dev, required before any public deployment.
