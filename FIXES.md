## Fix 1 — `fast_info.get()` Silent Zero Bug
**File:** `backend/routers/market_data.py`
**Severity:** Medium — corrupts DB cache with 0.0 on empty history fallback
**Agent:** Connectivity (Agent 2)

### Problem
`fast_info` is a `LazyLoadingDict` in yfinance. Calling `.get('last_price', 0.0)` does
not trigger lazy-loading and silently returns the default `0.0` instead of the real price.
This poisons the market data cache whenever Yahoo Finance returns an empty history
(market closure, rate limiting, bad ticker).

```python
# BEFORE — broken
if data.empty:
    info = ticker.fast_info
    return round(info.get('last_price', 0.0), 2)  # always returns 0.0
```

```python
# AFTER — fixed
if data.empty:
    info = ticker.fast_info
    try:
        last_price = info.last_price          # attribute access triggers lazy-load
        if last_price and last_price > 0:
            return round(float(last_price), 2)
    except Exception:
        pass
    raise ValueError(f"Empty history and no fast_info price for {data_type}")
```

## Fix 2 — Add Price Cache to `get_current_price()`
**File:** `backend/routers/portfolio_allocation.py`
**Severity:** High — causes 3.0s response time on `/api/portfolio-allocation/calculate`
**Agent:** Page Load (Agent 1)

### Problem
`get_current_price(symbol)` calls `yf.Ticker(symbol).info` (live HTTP) for every ETF
with no caching. On a 9-ETF portfolio this means 9 serial blocking HTTP calls per request.

```python
# BEFORE — no cache, 9 live HTTP calls per request
def get_current_price(symbol: str) -> float:
    ticker = yf.Ticker(symbol)
    info = ticker.info
    price = info.get('regularMarketPrice', ...)
    return float(price)
```

```python
# AFTER — 15-min in-memory cache + static fallbacks
import time

_price_cache: dict = {}
_PRICE_CACHE_TTL = 900  # 15 minutes

STATIC_PRICES = {
    "JEPI": 59.0, "JEPQ": 55.0, "SCHD": 28.0,
    "VYM": 125.0, "SGOV": 100.5, "QQQ": 490.0,
    "VUG": 360.0, "VOOG": 310.0, "SCHG": 105.0,
}

def get_current_price(symbol: str) -> float:
    now = time.time()
    cached = _price_cache.get(symbol)
    if cached and now - cached["ts"] < _PRICE_CACHE_TTL:
        return cached["price"]
    try:
        price = yf.Ticker(symbol).info.get(
            "regularMarketPrice", STATIC_PRICES.get(symbol, 100.0)
        )
    except Exception:
        price = STATIC_PRICES.get(symbol, 100.0)
    _price_cache[symbol] = {"price": float(price), "ts": now}
    return float(price)
```

## Fix 3 — Add Historical Performance Cache
**File:** `backend/routers/portfolio_allocation.py`
**Severity:** Critical — causes 6.1s response time on `/api/portfolio-allocation/historical-performance` (breaches 5s rule)
**Agent:** Page Load (Agent 1)

### Problem
`get_etf_historical_data()` makes 4 serial yfinance calls per ETF (one per time period).
With 9 ETFs that is 36 blocking HTTP calls per request, taking ~6 seconds total.

```python
# BEFORE — 36 serial HTTP calls, no cache
def get_etf_historical_data(symbol: str, name: str) -> dict:
    return {
        "return_3yr":  get_annualized_return(symbol, 3),
        "return_5yr":  get_annualized_return(symbol, 5),
        "return_10yr": get_annualized_return(symbol, 10),
        "return_20yr": get_annualized_return(symbol, 20),
    }
```

```python
# AFTER — 24-hour cache (historical returns don't change intraday)
_perf_cache: dict = {}
_PERF_CACHE_TTL = 86400  # 24 hours

def get_etf_historical_data(symbol: str, name: str) -> dict:
    now = time.time()
    cached = _perf_cache.get(symbol)
    if cached and now - cached["ts"] < _PERF_CACHE_TTL:
        return cached["data"]
    data = {
        "return_3yr":  get_annualized_return(symbol, 3),
        "return_5yr":  get_annualized_return(symbol, 5),
        "return_10yr": get_annualized_return(symbol, 10),
        "return_20yr": get_annualized_return(symbol, 20),
    }
    _perf_cache[symbol] = {"data": data, "ts": now}
    return data
```

## Fix 4 — Parallelize yfinance Calls with ThreadPoolExecutor
**File:** `backend/routers/portfolio_allocation.py`
**Severity:** High — complements Fix 2 & 3; reduces cold-cache latency from ~6s to ~0.8s
**Agent:** Page Load (Agent 1)

### Problem
Even with caching, the first (cold) request still makes all yfinance calls serially.
Since yfinance calls are I/O-bound, they can be parallelized easily.

```python
# AFTER — parallel fetches on cache miss
from concurrent.futures import ThreadPoolExecutor

# In get_historical_performance():
with ThreadPoolExecutor(max_workers=9) as executor:
    results = list(executor.map(
        lambda etf: get_etf_historical_data(etf["symbol"], etf["name"]),
        all_etfs
    ))

# In calculate_portfolio_allocation():
symbols = [etf["symbol"] for etf in all_etfs]
with ThreadPoolExecutor(max_workers=9) as executor:
    prices = list(executor.map(get_current_price, symbols))
```

## Fix 5 — Add Missing DB Indexes
**File:** `backend/models.py`
**Severity:** Low now, High at scale — full table scans on every filtered query
**Agent:** Page Load (Agent 1)

### Problem
Three columns used in WHERE clauses have no index. Currently harmless on an empty DB
but will degrade linearly as data grows (expenses, holdings, snapshots).

```python
# BEFORE
is_recurring = Column(Integer, default=0)
account_id   = Column(Integer, ForeignKey("brokerage_accounts.id"), nullable=False)
snapshot_date = Column(DateTime(timezone=True), nullable=False)

# AFTER — add index=True
is_recurring  = Column(Integer, default=0, index=True)           # expenses table
account_id    = Column(Integer, ForeignKey(...), index=True)     # holdings table
account_id    = Column(Integer, ForeignKey(...), index=True)     # account_snapshots table
snapshot_date = Column(DateTime(timezone=True), index=True)      # account_snapshots table
```

## Fix 6 — Downgrade urllib3 to Silence OpenSSL Warning
**File:** `backend/venv` (package version)
**Severity:** Low — warning-only, connections still succeed
**Agent:** Connectivity (Agent 2)

### Problem
macOS ships LibreSSL 2.8.3 but `urllib3 v2` requires OpenSSL 1.1.1+. This produces a
noisy warning on every yfinance call and could cause subtle TLS edge cases.

```bash
# Fix — pin urllib3 to v1.x
~/Documents/Claude/my-app/backend/venv/bin/pip install "urllib3<2.0"
```
